# 常见问题

本章用于集中整理在部署和使用 HyperZoneLogin 时较常遇到的问题。遇到异常时，建议先对照本章判断问题属于哪一类，再结合 [`配置摘要`](./配置摘要.md) 和 [`问题反馈`](./问题反馈.md) 继续排查或整理反馈材料。

## 加入游戏后提示“登录失败：无效会话（请尝试重启游戏及启动器）”

### 现象说明

这是一个比较典型的认证链路异常提示，通常表示玩家当前使用的登录方式、会话状态或识别方式，和服务器已安装的验证模块不匹配。

根据当前项目文档可以确认，HyperZoneLogin 的处理重点在玩家进入代理后的识别、验证与转发流程。如果提示出现在**进入服务器之前**，通常应优先排查启动器、账号登录状态或上游会话本身，而不要直接判断为 HyperZoneLogin 故障。

### 常见原因

1. **未安装 `auth-offline`，但玩家使用离线方式进入**
   如果服务器只安装了在线验证相关模块，而玩家实际是通过离线启动方式接入，就可能在认证阶段出现会话不匹配。

2. **离线玩家使用了非常规 UUID 或特殊启动器接入方式**
   当前离线识别会受到 `offlinematch.conf` 中匹配规则影响。若启动器使用了不常见的 UUID 生成方式，服务器可能无法按预期识别该玩家。

3. **正版 / Yggdrasil 玩家会话已过期**
   如果玩家长时间未重新登录启动器，或游戏客户端保留了过期会话，进入服务器时就可能被上游验证链路判定为无效会话。

### 建议处理方法

#### 情况一：服务器需要支持离线玩家，但未安装离线模块

- 安装 `auth-offline` 模块；
- 首次启动后确认相关配置已正常生成；
- 再结合 [`模块介绍`](./模块介绍.md) 与 [`配置摘要`](./配置摘要.md) 检查你的登录方案是否完整。

#### 情况二：特定启动器或特定 UUID 规则无法识别

- 先确认是否只有某一类启动器会出现该问题；
- 不要只描述为“离线进不来”，而应补充具体启动器名称、下载链接、登录方式与复现步骤；
- 反馈时同时附带相关服务器日志，方便判断是否属于 `offlinematch.conf` 规则覆盖不足。

如果你需要进一步反馈，建议优先阅读 [`问题反馈`](./问题反馈.md)。

#### 情况三：正版玩家会话过期

- 完全退出游戏；
- 完全退出启动器；
- 重新登录启动器账号；
- 重新启动游戏后再次尝试进入服务器。

这是最常见、也最容易被忽略的一类原因。

### 建议反馈时附带的材料

如果以上步骤后仍无法解决，建议至少整理以下信息：

- HyperZoneLogin 主插件版本；
- 已安装的子模块列表；
- Velocity 与后端服务端版本；
- 玩家使用的启动器名称与下载链接；
- 玩家是正版登录、第三方 Yggdrasil 登录，还是离线登录；
- 相关报错日志与复现步骤。
