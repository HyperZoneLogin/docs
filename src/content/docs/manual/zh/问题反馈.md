---
title: "问题反馈"
---

当你在使用 HyperZoneLogin 过程中遇到问题，建议先做一次基础自检，再选择合适的反馈渠道。这样既能提高问题被定位的效率，也能避免把本来属于环境配置的问题误判成插件 Bug。

## 反馈前先自检

在提交问题之前，建议先确认以下内容：

- [ ] 你已经阅读过本手册对应章节；
- [ ] 你已经确认 Velocity 与后端服基础转发配置无误；
- [ ] 你知道自己安装了哪些模块；
- [ ] 你能提供复现步骤，而不只是“不能用”；
- [ ] 你已经查看过启动日志或报错信息；
- [ ] 如果问题与迁移相关，你已经备份过原始数据。

## 建议附带的信息

无论你是在 GitHub 还是 QQ 群反馈，尽量一次性提供下面这些信息：

1. **使用的版本**
   - `HyperZoneLogin` 主插件版本
   - 所有已安装子模块版本
   - Velocity 版本
   - 后端服务端版本

2. **部署结构**
   - 是否使用 Velocity
   - 是否使用 `modern forwarding`
   - 当前激活或安装的模块

3. **问题现象**
   - 玩家在哪一步出问题
   - 是首次登录失败、迁移失败、皮肤异常，还是跳服异常
   - 是否能稳定复现

4. **日志与配置**
   - 相关报错日志
   - 涉及的配置片段
   - 是否改动过默认配置

5. **复现步骤**
   - 玩家如何进入
   - 执行了什么命令
   - 预期结果是什么
   - 实际结果是什么

这些信息越完整，问题越容易被准确判断。

## 反馈提交通道

HyperZoneLogin目前接受从 [GitHub Issue](https://github.com/HyperZoneLogin/HyperzoneLogin/issues) 和QQ群 [832210691](https://qm.qq.com/q/GZWVfEyokS) 提交反馈，你可以选择自己熟悉的渠道。

GitHub Issue 界面提供了问题提交模板来帮助你填写所需信息，请按照提示逐项填写。在QQ群内提问时，也请提供上述的必需信息。

信息越完整，开发者就更能复现问题环境，问题越有可能被解决。登录框架的问题往往横跨代理、后端、数据库与认证服务。没有上下文，别人几乎无法准确判断。换句话说，**先阅读、先自检、再反馈**，会让你更快得到真正有用的帮助。

## 推荐模板

你可以直接按下面的格式整理问题：

```text
环境：Velocity 版本 / 后端版本 / HyperZoneLogin 版本 / 已安装模块
现象：发生了什么问题
步骤：如何稳定复现
日志：关键报错内容
配置：相关配置片段
已尝试：你已经做过哪些排查
```

如果你能把问题整理到这个程度，无论是发 Issue 还是去 QQ 群交流，效率都会高很多。
