# 快速上手

如果你现在的目标是：**先把 HyperZoneLogin 跑起来，再逐步细化配置**，那就按本章完成一套最小可用部署。

这份快速上手优先采用工作区 `ref/vc/` 中已经存在的完整示例思路：**Velocity 代理 + 真实后端等待区服务器 + 真实游戏区服务器**。这也是当前更稳妥、排障成本更低的入门方式。

## 先确认前置条件

开始前请先确认：

- [ ] 你已经准备好一台 Velocity 代理；
- [ ] 你至少准备了两个后端服：一个等待区服、一个正式游戏服；
- [ ] 你打算让玩家通过代理地址进入，而不是直连后端；
- [ ] 你已经看过 [`服务器基础配置`](./服务器基础配置.md)。

尤其是 `modern forwarding`、`forwarding.secret`、后端 `online-mode` 这些基础项，**本章不会重复展开配置片段**。如果这里还没有配置好，请先回到 [`服务器基础配置`](./服务器基础配置.md)。

## 推荐的最小拓扑

参考 `ref/vc/velocity.toml`，快速上手建议至少准备下面这两个后端：

1. `lobby`：作为认证等待区使用；
2. `play`：作为认证完成后的正式游戏区。

示例 `vc` 中也正是这样配置的：

- Velocity 的 `servers` 里存在 `lobby` 与 `play`；
- `try` 优先指向 `lobby`；
- `plugins/hyperzonelogin/backend-server.conf` 中：
  - `fallback-auth-server=lobby`
  - `post-auth-default-server=play`
  - `remember-requested-server-during-auth=true`

你不一定非要使用 `lobby`、`play` 这两个名字，但**Velocity 里的服务器名必须和 HyperZoneLogin 配置里的名字一致**，否则认证前后的跳转就会找不到目标服务器。

## 安装方式

对于第一次部署，建议直接使用发行页提供的 **单文件发行 jar**（常见文件名会带 `-all.jar`），然后把它直接放进 Velocity 的 `plugins/` 目录。

推荐顺序如下：

1. 下载 HyperZoneLogin 的发行 jar；
2. 把 jar 放进代理服的 `plugins/`；
3. 启动一次 Velocity，让 `plugins/hyperzonelogin/` 配置目录自动生成；
4. 按照上面的后端命名，把 `backend-server.conf` 调整为等待区服和游戏区服；
5. 再检查你的 Velocity 服务器列表和后端服是否都已启动。

## 为什么快速上手优先不用 limboAPI

虽然某些部署会考虑把 `limboapi` 作为认证等待区方案，但对于“先跑通”的目标来说，不建议把它当成第一选择。

原因很简单：**快速上手更强调稳定、可观察、容易排障。** 相比之下，直接准备一个真实后端等待区服务器，和 `ref/vc/` 一样使用 `fallback-auth-server` / `post-auth-default-server` 的方式，通常更容易验证链路是否正常。

因此，本章建议：

- 初次部署优先使用真实 backend 等待区服务器；
- `limboapi` 方法不能保证完全稳定；
- 如果你后续确实要研究该方案，建议在已经跑通 backend 方案后再单独测试。

## 完成后的验收方式

当你完成代理、后端、插件放置与基础配置后，建议直接启动 Minecraft 客户端，通过代理地址加入服务器进行验收。

如果链路正常，通常可以按下面方式判断是否已经配置成功：

1. 玩家先进入等待区后端服；
2. 在等待区服务器里看到的玩家名表现为 `HZLxxxxxx`（`xxxxxx` 为一串随机数字）；
3. 认证完成并进入正式游戏区服务器后，玩家名恢复为正常玩家名称。

如果你观察到的是这类结果，通常说明：

- 代理层接管了认证流程；
- 等待区与游戏区切换正常；
- HyperZoneLogin 的基础链路已经跑通。

如果没有出现上述现象，优先回查以下内容：

- [`服务器基础配置`](./服务器基础配置.md)：确认 `modern forwarding` 与后端代理设置；
- `velocity.toml`：确认后端服务器名与 `try` 顺序；
- `backend-server.conf`：确认等待区服与目标服名称填写正确。

## 跑通之后下一步看什么

当你已经完成快速上手后，建议继续按下面顺序阅读：

1. [`模块介绍`](./模块介绍.md)：确定你到底需要哪些模块；
2. [`基础安装`](./基础安装.md)：了解更完整的安装与首次启动检查项；
3. [`配置摘要`](./配置摘要.md)：如果你还要接入自己的皮肤站、自定义 Entry、调整数据库或离线识别策略，这一章最值得继续看。

如果你只是想先验证“能不能跑”，那到这里已经足够；如果你准备做正式部署，请不要跳过后续章节。
